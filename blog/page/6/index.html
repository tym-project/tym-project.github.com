Lutter Contre Le SPAM Par “Sms”

Dec 30th, 2009

“rappelle moi au 0899XXXXXX je suis impatiente !”. C’est ce genre de message que tout un chacun a déjà reçu, ou recevra certainement sur son mobile. Inutile de préciser que c’est un SPAM visant à vous faire dépenser vos précieux $$$ !

La première solution pour “lutter” contre ce système c’est de ne pas tomber dans le piège… au moins !
Mais vous pouvez être encore plus malin et éviter de recevoir encore des messages de ce type : il suffit de répondre “STOP” à l’expéditeur du message… la loi oblige alors l’expéditeur à ne plus vous envoyer de message !

J’ai pour ma part testé cette manipulation à plusieurs reprises, avec succès !
Enfin, si vous vous sentez l’âme d’un bon samaritain, et dans le cas du SPAM tout le monde devrait se sentir concerné, vous pouvez dénoncer l’expéditeur ! Pour cela transmettez le message au “33700”, c’est un service du gouvernement qui rassemble les “plaintes” et, dans le cas d’un grand nombre, engage des procédures… bye bye SPAMmer !
Ces deux dernières manipulations ne vous coûteront pas plus cher que le prix d’un sms non surtaxé !

Bon nettoyage ;-)

Pathping : Le Ping Mutant en Traceroute !

Sep 21st, 2009

Pour rappel l’outil [man]traceroute[/man] permet de voir le chemin suivit par un message sur le réseau. C’est donc une commande très pratique pour diagnostiquer et résoudre des problèmes de routage.

Vous connaissez certainement ping qui permet de vérifier la connectivité avec une machine (via le protocole ICMP). Il existe une commande qui rassemble ces deux éléments : pathping. Cet outil est disponible sous Windows nativement, et via certains paquets sous Linux (selon la distribution).

Mais comme une image vaut mieux qu’un long discours…voici (cliquez pour agrandir) :

Comme on peut le voir sur la capture, pathping détermine l’itinéraire suivit par le paquet, ainsi que le temps passé sur chaque hôte intermédiaire…pratique pour détecter d’éventuels points de congestion dans le réseau !
